# 自托管说明

[English](./SELF_HOSTING.md) | **简体中文**

## 概览

Prism Translate 支持本地运行，也支持部署到你自己的托管环境。当前架构基于“浏览器侧管理 Provider 配置 + 服务端 API 执行翻译与模型发现”这一模式。

## 本地开发

常见本地流程：

```bash
npm install
npm run dev
```

可选运行时覆盖：

```bash
cp .env.example .env.local
```

当前唯一明确记录的环境变量是：
- `REQUEST_TIMEOUT_MS`

Provider 凭证默认不是从环境变量读取，而是由用户在应用 UI 中配置。

## 托管部署预期

本仓库兼容标准 Next.js 托管环境，包括 Vercel。

需要明确的运行时事实：
- 翻译和模型发现都通过服务端 API 路由执行
- 浏览器用户会在操作过程中把 Provider 配置发送给这些 API 路由
- 超长上游请求是否成功，取决于你的平台超时限制

## Vercel 说明

当前代码路径已经针对 Vercel 兼容部署做了适配，但仍然存在实际限制：
- 路由执行时间不是无限的
- 上游 Provider 过慢时，仍可能触发平台执行时长限制
- 请求超时会被限制在 Vercel 路由时长上限以内

如果你部署在 Vercel 之外，请特别检查平台默认的超时、日志和可观测性配置。

## 自定义端点与 Ollama

某些 Provider 方案需要额外注意：
- `ollama` 默认的本地地址在托管环境中不可直接访问，除非你额外暴露一个可达端点
- 自定义或 OpenAI-compatible 端点必须能够被部署后的服务端运行时访问
- Provider 专属 headers、账号信息等配置，仍然要通过应用的配置模型由用户自行提供

## 哪些场景适合这种架构

比较适合：
- 个人使用
- 可信任的内部团队
- 自托管的翻译对比工作台
- 能接受当前凭证模型的受控环境

不太适合：
- 运维方不应接触用户凭证的公开多租户托管场景
- 需要把部署者完全隔离在 secrets 之外的环境
- 会记录请求体或把原始载荷转发给第三方可观测性服务的部署环境

## 运维建议

- 禁用或严格限制请求体日志
- 把导出的 `.prism` 文件视为敏感文件
- 如果有凭证误入 issue 或日志，立即轮换
- 检查反向代理、CDN 与可观测性链路中是否存在 secrets 泄露风险

关于凭证流向与信任边界的精确定义，请看 [TRUST_MODEL.zh.md](TRUST_MODEL.zh.md)。
