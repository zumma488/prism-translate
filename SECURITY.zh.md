# 安全策略

## 支持版本

Prism Translate 目前以 `main` 分支上的最新代码为主要维护目标。安全修复通常会尽力优先基于 `main` 处理。

## 漏洞报告

本仓库的安全问题通过公开 GitHub Issues 报告。

发布前请先完成以下脱敏：

- 删除或轮换真实 API Key
- 不要附带包含真实凭证的导出 `.prism` 文件
- 不要粘贴包含 secrets 的原始请求载荷或日志
- 对 Provider headers、tokens、账号标识和请求体做脱敏处理

如果你无法在不暴露 secrets 的前提下描述问题，请先轮换受影响的凭证，再提交脱敏后的报告。

反馈时请尽量包含：

- 问题的清晰描述
- 复现步骤或 PoC
- 影响评估
- 如果你已经有初步缓解建议，也欢迎一并提供

## 重点范围

请重点关注以下方向：

- Provider 凭证处理
- 可能包含敏感信息的导出 `.prism` 文件
- 浏览器侧存储的 Provider 配置
- 服务端/API 代理行为与请求处理
- 可能记录传输中 secrets 的请求体日志或第三方可观测性配置

## 当前安全模型

- Provider 配置由用户保存在浏览器中。
- 导出的 `.prism` 文件是明文 JSON。
- 在拉取模型和执行翻译时，浏览器会把 Provider 配置发送给应用的 API 路由。
- 服务端运行时再使用这些配置去调用上游 Provider。
- 任何部署并运营该应用的人，都处于这些凭证经过时的信任边界内。

## 披露原则

由于本仓库采用公开问题反馈，请只公开脱敏后的复现细节和影响说明，不要在漏洞讨论中发布 secrets。
