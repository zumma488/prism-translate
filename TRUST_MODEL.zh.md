# 信任模型

[English](./TRUST_MODEL.md) | **简体中文**

## 为什么需要这份文档

Prism Translate 不是一个零信任的托管式密钥管理产品。这份文档用于说明 Provider 凭证当前存放在哪里、如何流动，以及在现有架构下你必须信任谁。

## 核心模型

当前的信任模型是：
- 用户把 Provider 配置保存在自己的浏览器中
- 浏览器在需要时把 Provider 配置发送给应用 API 路由
- 服务端运行时再使用这些配置去调用上游 Provider
- 导出的 `.prism` 文件是明文 JSON

这意味着：如果你部署了一个托管实例，部署者/运维者本身就在信任边界内。

## 凭证流向

1. 用户在浏览器 UI 中配置 Provider 凭证。
2. 这些设置保存在浏览器本地。
3. 当用户拉取模型或执行翻译时，浏览器会把 Provider 配置发送到：
   - `app/api/providers/models`
   - `app/api/translate/stream`
4. 服务端运行时接收这些配置，并用它们去调用所选的上游 Provider。
5. 最终结果再返回浏览器。

## 存储现实

- 浏览器侧设置存储是本地、用户自管的，不是企业级密钥存储方案。
- 导出的 `.prism` 文件是明文 JSON。
- 旧版加密 `.prism` 导入已经不再支持。

## 信任边界

你必须信任：
- 运行浏览器的本地机器
- 该应用的部署者/运维者
- 任何可以查看请求或日志的基础设施组件

你不应假设：
- 部署者与用户凭证之间存在密码学级别的隔离
- 如果平台默认记录请求体，请求载荷依然天然安全
- 这种架构适合对密钥隔离要求很高的公开多租户托管场景

## 可观测性与日志风险

需要特别小心：
- 请求体日志
- 会回显请求载荷的错误日志
- 第三方可观测性或 tracing 工具
- 会捕获原始流量的反向代理与网关

如果这些地方配置不当，就可能暴露 Provider 凭证。

## 更安全的使用方式

更适合：
- 本地使用
- 个人自托管
- 可信任的内部环境

风险更高：
- 公开/共享托管部署
- 不允许运维方接触用户凭证的环境
